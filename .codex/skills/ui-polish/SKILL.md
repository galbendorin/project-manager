# UI Polish

Use this skill when a PM Workspace screen feels crowded, awkward, generic, or inconsistent across desktop and mobile.

## Focus
- hierarchy and spacing
- mobile ergonomics
- clear primary actions
- empty, loading, and error states
- concise product copy

## Inputs
- target screen or file
- screenshot or route when available
- any explicit product constraint from the user

## Output
- the smallest defensible UI change set
- short note on what improved
- desktop and mobile verification notes

## Process
1. Inspect the current screen and name the top three UX problems.
2. Fix structure before decoration.
3. Prefer removing clutter over adding new elements.
4. Keep authenticated screens compact and functional.
5. Verify the result at desktop and phone widths.

## PM Workspace Defaults
- Preserve the existing visual language unless the task asks for a redesign.
- Avoid generic dashboard cards that repeat the same message in multiple places.
- Keep titles, labels, and CTA copy literal.
- For mobile, make actions reachable before making the surface more expressive.
